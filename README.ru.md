# 🛡️ claude-code-haiku-guard

Любой, кто активно пользуется Claude Code, рано или поздно устает подтверждать безобидные команды. Большинство людей начинают с добавления исключений в список разрешенных команд, а некоторые в итоге включают --dangerously-skip-permissions. Отчеты об инцидентах, ссылки на которые приведены ниже, наглядно показывают, насколько это рискованно.

Идея этого репозитория простая: для очевидно безопасных команд использовать правила, а для более сложных случаев подключать через API быструю и недорогую модель вроде Haiku. В результате большая часть рутинных команд проходит автоматически, а пользователю достаются только те случаи, которые выглядят опасно или недостаточно понятны.

Технически это хук для Claude Code, который классифицирует полную Bash-команду по риску. По умолчанию LLM-шаг работает через OpenRouter.

> [English version →](README.md)

## ⚖️ Что происходит с командами

| Риск | Что происходит |
|------|----------------|
| `none` / `low` | команда проходит без диалога |
| `medium` | Haiku решает, пропустить команду или показать диалог |
| `high` / `critical` | всегда показывается диалог |

Примеры реальных инцидентов, когда ИИ использовал широкие разрешения и производил деструктивные действия, собраны в [INCIDENTS.ru.md](INCIDENTS.ru.md).

## 🔍 Как принимается решение

1. Сначала команда проверяется по набору правил на регулярных выражениях.
2. Обертки интерпретаторов вроде `python -c`, `bash -c` и `powershell -Command` не опускаются ниже уровня `medium`.
3. Для команд уровня `medium` и для незнакомых случаев вызывается Haiku с учетом команды, текущего каталога и необязательной проектной конфигурации.
4. Решения кэшируются по полной команде и `cwd`.

Если ключ OpenRouter не настроен, сеть недоступна или запрос к модели завершился ошибкой, команды уровня `medium` переводятся в диалог. На Windows при этом появляется окно с пояснением. Для `high` и `critical` диалог обязателен всегда.

## 🚀 Быстрый старт

Подробная инструкция есть в [SETUP.md](SETUP.md). Короткая версия такая:

1. Скопируйте `hook/haiku_guard.py` в `~/.claude/hooks/haiku_guard.py`.
2. Положите ключ OpenRouter (`sk-or-...`) в `~/.openrouter_key`.
3. Добавьте `PreToolUse`-хук из [examples/settings.json](examples/settings.json) в `~/.claude/settings.json`.
4. Уберите из списка разрешений широкие правила вида `Bash(<что-то> *)`.
5. Перезапустите Claude Code.

## ⚙️ Настройка

Необязательная конфигурация лежит в `~/.claude/hooks/haiku_guard.config.json`:

```json
{
  "critical_files": ["CLAUDE.md", "*.csproj", "MyProject.sln"],
  "critical_dirs": [".claude/", "src/core/", "migrations/"],
  "development_processes": ["dotnet", "node", "python"]
}
```

Пропущенные поля берутся из `DEFAULT_CONFIG` в [hook/haiku_guard.py](hook/haiku_guard.py).

Переменные окружения:

- `HAIKU_GUARD_OPENROUTER_KEY`
- `HAIKU_GUARD_OPENROUTER_KEY_FILE`
- `HAIKU_GUARD_MODEL`
- `HAIKU_GUARD_TIMEOUT`

По умолчанию используется `anthropic/claude-haiku-4.5`. Через `HAIKU_GUARD_MODEL` можно выбрать и другую модель OpenRouter, но отдельно прогонялся только базовый вариант. После смены модели лучше заново запустить сетевые тесты.

## 🧪 Тесты

```bash
cd tests
python test_classifier.py
python test_haiku_decision.py
python test_interpreter_destructive.py
```

`test_classifier.py` полностью офлайн. Два набора с Haiku требуют ключ OpenRouter и при его отсутствии завершаются со статусом SKIP, а не с ошибкой.

## ⚠️ Ограничения

- Правила и примеры в первую очередь рассчитаны на Windows и Git Bash.
- Контракт хуков здесь привязан к Claude Code.
- Решения для `medium` зависят от OpenRouter. Без ключа такие команды будут уходить в диалог.
- Широкие Bash-правила в настройках Claude сводят пользу этого хука почти к нулю, поэтому их лучше убрать и из глобальных, и из проектных настроек.
- Кэш привязан к полной команде и `cwd`, поэтому та же команда в другой директории будет оценена заново.
