# 🛡️ claude-code-haiku-guard

**Страж опасных Bash-команд для Claude Code на базе Claude Haiku. Рутинные команды одобряются молча, подтверждение всплывает только на реальном риске — чтобы можно было работать в режиме «Edit automatically» и при этом не ловить `rm -rf` проекта от агента.** 🔥

> 🇬🇧 [English version →](README.md)

## 🎯 Кому подойдёт

Пользователи Claude Code, которые:
- работают в режиме **«Edit automatically»** (правки файлов идут молча), и
- устали от **постоянных диалогов подтверждения** на каждый `ls` / `git status`, но
- не хотят тупо прописать `Bash(*)` / `Bash(git *)` и смотреть как агент молча делает `git reset --hard` или `rm -rf`.

## ⚠️ Зачем это нужно

Стандартный allow-лист Claude Code использует glob-паттерны. Правило `Bash(echo *)` матчит **любую** команду начинающуюся с `echo`, включая `echo ok && rm -rf .git` — потому что `*` матчит пробелы, `&&` и пайпы. Каждый «безобидно выглядящий» wildcard в allow-листе — потенциальный обход для деструктивных операций.

**Да, так действительно бывает** 😱 — см. [INCIDENTS.ru.md](INCIDENTS.ru.md): список публичных инцидентов, где AI-агенты (Claude Code, Cursor, Codex) молча выполняли `rm -rf`, `git reset --hard` и деструктивные циклы PowerShell в реальных проектах.

Этот хук заменяет allow-лист на **классификацию по риску**:

| Риск | Что происходит |
|------|----------------|
| 🟢 `none` / `low`   | Молча одобряется (без диалога) |
| 🟡 `medium`         | Haiku читает команду в контексте и решает: рутина → молча пропустить, подозрительное → диалог |
| 🔴 `high` / `critical` | Всегда диалог — Haiku не может переопределить |

Классификатор начинает с быстрых regex-правил (~80 известных паттернов: `rm -rf`, `git push --force`, `kill` и т.д.), для сложного отправляет в Haiku, и **фиксирует минимум `medium` для интерпретаторов** (`python -c`, `powershell -Command`, `bash -c`, ...) — чтобы LLM не мог понизить запуск произвольного кода.

## Как это работает

```
                  ┌─────────────────────────────────────────────────┐
  Bash-команда ──►│ 1. Правила-классификатор  (regex, ~80 паттернов)│
                  │ 2. Floor для интерпретаторов (-c / -e минимум medium)│
                  │ 3. LLM-классификатор       (Haiku для сложного) │
                  └─────────┬───────────────────────────────────────┘
                            │
             none/low ──────┼──► allow (молча)
             medium ────────┼──► Haiku решает с учётом cwd и конфига
                            │    → allow  (рутина)
                            │    → ask    (подозрительно → диалог)
             high/critical ─┴──► ask (всегда диалог, Haiku не переопределяет)
```

**Fail-closed по дизайну.** Нет ключа OpenRouter, таймаут сети, ошибка LLM → диалог, а не тихое разрешение.

**Безопасный кэш.** Решения ключуются по полной команде + `cwd`, чтобы одобрение одной команды не пролезло для другой с тем же префиксом.

**Prompt caching включён.** System-промпт отправляется с `cache_control: ephemeral` — Anthropic кэширует его (TTL 5 минут). Запросы подряд платят ~10% от стоимости системной части вместо полной цены.

## Установка за ~60 секунд

Подробно — в [SETUP.md](SETUP.md). Коротко:

1. Скопируй `hook/haiku_guard.py` в `~/.claude/hooks/haiku_guard.py`
2. Положи ключ OpenRouter (`sk-or-...`) в `~/.openrouter_key`
3. Примерджи `examples/settings.json` в `~/.claude/settings.json` — ключевой кусок это `PreToolUse` хук на `Bash`
4. Убери широкие `Bash(X *)` wildcard-паттерны из allow-листа
5. Перезагрузи Claude Code

## Настройка

Хук работает из коробки. Чтобы научить его защищать твои ключевые артефакты проекта (файлы и папки которые нельзя молча удалять/двигать) — создай `~/.claude/hooks/haiku_guard.config.json`:

```json
{
  "critical_files": ["CLAUDE.md", "*.csproj", "MyProject.sln"],
  "critical_dirs": [".claude/", "src/core/", "migrations/"],
  "development_processes": ["dotnet", "node", "python"]
}
```

Пропущенные поля берутся из дефолтов — смотри `DEFAULT_CONFIG` в [hook/haiku_guard.py](hook/haiku_guard.py).

Env-переменные:
- `HAIKU_GUARD_OPENROUTER_KEY` — ключ inline (перекрывает файл)
- `HAIKU_GUARD_OPENROUTER_KEY_FILE` — альтернативный путь к файлу ключа
- `HAIKU_GUARD_MODEL` — id модели в OpenRouter (по умолчанию `anthropic/claude-haiku-4.5`)
- `HAIKU_GUARD_TIMEOUT` — таймаут LLM-вызова в секундах (по умолчанию `15`)

> ℹ️ **Альтернативные модели.** Если Claude недоступен (регион, предпочтения, стоимость) — через `HAIKU_GUARD_MODEL` подставляется Mistral / DeepSeek / Qwen / Llama. Список кандидатов — в [SETUP.md § Choosing a model](SETUP.md#choosing-a-model). Сравнительное тестирование пока не проводилось — часть пограничных кейсов может начать классифицироваться иначе.

## Стоимость

Рабочий день разработчика — ~50-100 вызовов Haiku, по 5 токенов на выход. OpenRouter-цена на Haiku 4.5 даёт меньше $0.10 в день.

## Ограничения

- **Заточен под Windows.** Правила и примеры рассчитаны на Windows + Git Bash. На Linux/macOS классификатор работает, но часть PowerShell-правил лишние.
- **Только Claude Code.** Контракт PreToolUse / PermissionRequest специфичен для Claude Code. У Cursor, Codex, Continue и других — свои API.
- **Зависимость от OpenRouter.** Для Haiku-решения нужен живой ключ. Без него страж всё ещё классифицирует правилами и на каждый `medium+` покажет диалог — безопасно, но больше кликов.
- **Кэш привязан к `cwd`.** Если одна и та же команда выполняется в другой директории — cache miss, Haiku спросится заново. Это намеренно — фикс бага «повтора одобрения в другом контексте».
