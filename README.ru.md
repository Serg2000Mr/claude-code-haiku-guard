# 🛡️ claude-code-haiku-guard

**Страж опасных Bash-команд для Claude Code на базе Claude Haiku. Рутинные команды одобряются молча, подтверждение всплывает только на реальном риске — чтобы можно было работать в режиме «Edit automatically» и при этом не ловить `rm -rf` проекта от агента.**

> [English version →](README.md)

## Кому подойдёт

Пользователи Claude Code, которые:
- работают в режиме **«Edit automatically»** (правки файлов идут молча), и
- устали от **постоянных диалогов подтверждения** на каждый `ls` / `git status`, но
- не хотят просто прописать `Bash(*)` / `Bash(git *)` и наблюдать, как агент молча делает `git reset --hard` или `rm -rf`.

## ⚠️ Зачем это нужно

Стандартный список разрешений Claude Code устроен на glob-шаблонах. Правило `Bash(echo *)` совпадает с **любой** командой, начинающейся с `echo`, включая `echo ok && rm -rf .git` — потому что `*` ловит пробелы, `&&` и каналы. Каждый «безобидно выглядящий» шаблон в списке разрешений — потенциальный обход для деструктивных операций.

**Да, так действительно бывает** — см. [INCIDENTS.ru.md](INCIDENTS.ru.md): список публичных инцидентов, где AI-агенты (Claude Code, Cursor, Codex) молча выполняли `rm -rf`, `git reset --hard` и деструктивные циклы PowerShell в реальных проектах.

Этот перехватчик заменяет список разрешений на **классификацию по риску**:

| Риск | Что происходит |
|------|----------------|
| `none` / `low`   | Молча одобряется (без диалога) |
| `medium`         | Haiku читает команду в контексте и решает: рутина → молча пропустить, подозрительное → диалог |
| `high` / `critical` | Всегда диалог — Haiku не может переопределить |

Классификатор сначала пробует быстрые правила на регулярных выражениях (~80 известных шаблонов: `rm -rf`, `git push --force`, `kill` и т.д.), сложные команды отправляются в Haiku, а для интерпретаторов (`python -c`, `powershell -Command`, `bash -c`, ...) жёстко фиксируется **минимум среднего риска** — чтобы модель не могла понизить запуск произвольного кода.

## Как это работает

```
                  ┌──────────────────────────────────────────────────────┐
  Bash-команда ──►│ 1. Классификатор по правилам  (regex, ~80 шаблонов)  │
                  │ 2. Минимум для интерпретаторов (-c / -e → не ниже    │
                  │    среднего риска)                                   │
                  │ 3. Классификация через LLM    (Haiku, для сложных)   │
                  └─────────┬────────────────────────────────────────────┘
                            │
             нет / низкий ──┼──► разрешить (молча)
             средний ───────┼──► Haiku решает с учётом рабочего каталога
                            │    → разрешить (рутина)
                            │    → спросить  (подозрительно → диалог)
             высокий/       ─┴──► спросить (всегда диалог, Haiku
             критический          не переопределяет)
```

**Безопасный отказ.** Нет ключа OpenRouter, срок ожидания истёк, ошибка LLM — показываем диалог, а не тихо разрешаем.

**Безопасный кэш.** Ключ решения строится из полной команды и рабочего каталога, чтобы одобрение одной команды не утекло для другой с тем же префиксом.

**Кэширование системного промпта включено.** Системный промпт отправляется с `cache_control: ephemeral` — Anthropic кэширует его (срок жизни 5 минут). Запросы подряд платят около 10 % от стоимости системной части вместо полной цены.

## 🚀 Установка за ~60 секунд

Подробно — в [SETUP.md](SETUP.md). Коротко:

1. Скопируйте `hook/haiku_guard.py` в `~/.claude/hooks/haiku_guard.py`
2. Положите ключ OpenRouter (`sk-or-...`) в `~/.openrouter_key`
3. Добавьте блок `hooks` из `examples/settings.json` в свой `~/.claude/settings.json` — главное там перехватчик `PreToolUse` с совпадением `Bash`
4. Уберите из списка разрешений широкие шаблоны `Bash(X *)`
5. Перезагрузите Claude Code

## Настройка

Перехватчик работает сразу. Чтобы научить его защищать ключевые файлы и каталоги проекта (те, что нельзя молча удалять или двигать), создайте `~/.claude/hooks/haiku_guard.config.json`:

```json
{
  "critical_files": ["CLAUDE.md", "*.csproj", "MyProject.sln"],
  "critical_dirs": [".claude/", "src/core/", "migrations/"],
  "development_processes": ["dotnet", "node", "python"]
}
```

Пропущенные поля берутся из значений по умолчанию — см. `DEFAULT_CONFIG` в [hook/haiku_guard.py](hook/haiku_guard.py).

Переменные окружения:
- `HAIKU_GUARD_OPENROUTER_KEY` — ключ прямо в переменной (перекрывает файл)
- `HAIKU_GUARD_OPENROUTER_KEY_FILE` — альтернативный путь к файлу ключа
- `HAIKU_GUARD_MODEL` — идентификатор модели в OpenRouter (по умолчанию `anthropic/claude-haiku-4.5`)
- `HAIKU_GUARD_TIMEOUT` — срок ожидания ответа LLM в секундах (по умолчанию `15`)

> **Альтернативные модели.** Если Claude недоступен (регион, предпочтения, стоимость) — через `HAIKU_GUARD_MODEL` подставляется Mistral / DeepSeek / Qwen / Llama. Список кандидатов — в [SETUP.md § Choosing a model](SETUP.md#choosing-a-model). Сравнительное тестирование пока не проводилось — часть пограничных кейсов может начать классифицироваться иначе.

## Стоимость

Рабочий день разработчика — примерно 50-100 вызовов Haiku, по 5 токенов на ответ. Цена Haiku 4.5 в OpenRouter даёт меньше $0.10 в день.

## Ограничения

- **Заточено под Windows.** Правила и примеры рассчитаны на Windows и Git Bash. На Linux и macOS классификатор тоже работает, но часть правил для PowerShell становится лишней.
- **Только Claude Code.** Контракт событий `PreToolUse` и `PermissionRequest` свой у Claude Code. У Cursor, Codex, Continue и других — собственные интерфейсы, код придётся адаптировать.
- **Зависимость от OpenRouter.** Для решения через Haiku нужен живой ключ. Без него страж всё равно классифицирует команды по правилам и показывает диалог на каждый средний и выше уровень — безопасно, но кликов больше.
- **Кэш привязан к `cwd`.** Если та же команда выполняется в другой директории, промаха по кэшу не избежать — Haiku будет спрошен заново. Это намеренное решение, оно закрывает ошибку «одобрение из одного контекста подставилось в другой».
